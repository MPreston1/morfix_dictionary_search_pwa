Used only to obtain an anonymous auth token, the same way Morfix's own frontend does. Not a secret — this key just identifies the Firebase project. Pealim needs no key at all — it's a plain page fetch.
When checked, Pealim searches always use the local backup below — no live request is made, and results won't include anything crawled since your last backup.

Crawls pealim.com's full "All words" listing (~9,000 entries) plus full conjugation tables for every verb, and stores it all in this browser's IndexedDB so Pealim searches work offline. Takes a while the first time (many small requests through the same CORS proxy chain above) — safe to pause and resume anytime, including after closing the app. Use the checkbox above to actually search it; use "Download backup as JSON" to save a copy (e.g. to move it to another browser or device), and "Import backup from JSON" to load a previously downloaded copy instead of re-crawling from scratch — importing merges into whatever's already cached here.
Both dictionaries search through a chain of CORS routes: corsproxy.io first, then a direct request, then allorigins and codetabs as fallbacks. Morfix additionally requires its bearer token to be forwarded, which only corsproxy.io and the direct attempt can carry — so if corsproxy.io ever goes down or gets rate-limited, Morfix searches may fail even while Pealim's still work fine.
